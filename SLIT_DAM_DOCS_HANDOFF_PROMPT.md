## 基本方針

- スリットダム画面設計書だけを更新する
- 1セクションずつ進める
- 実アプリの現行UIを正として更新する
- A/B 両方を確認する
- 画像は新しいスクショをベースにする
- `draw.io` ローカルアプリで番号を振る
- 表示用画像は `.drawio.svg`
- 編集元として `.drawio` を残す
- スクショ `.png` も残す
- Markdown には作業メモを書かない
- 表は既存カラムのまま更新する

## 番号と表のルール

- 画像の番号は表の `No` と必ず一致させる
- 既存画像の付番スタイルに寄せる
- 番号は対象のすぐ近くに置く
- 文字の上に被せない
- 遠すぎて対応先が分からなくならないようにする
- 結果表のような密な画面では、行ラベル・変数・単位・代表値を分けて番号を振ってよい
